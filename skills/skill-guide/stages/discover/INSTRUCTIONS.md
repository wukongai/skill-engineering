# Discover 阶段

当用户有一个能力想法,但还不确定应该做成什么产物时,使用这个阶段。

1. 收集 2-5 个应该触发该能力的用户说法。
2. 收集 2-5 个不应该触发该能力的反例。
3. 读取 `references/selection-matrix.md`。
4. 推荐一种选择:新 skill、复用已有 skill、plugin/runtime、script、project doc、profile entry、archive/remove 或 no-op。
5. 如果推荐创建 skill,说明它是 atomic、orchestrator、router 还是 adapter。
6. 涉及 global 暴露或 plugin 启用时,先停下让用户确认。
