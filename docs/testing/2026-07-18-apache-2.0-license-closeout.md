## 构建与分发元数据

首次运行 `python3 -m build` 时，当前 Python 环境缺少 `build` 模块；首次隔离 `uv build` 又因沙箱不能解析 PyPI，二者均属于构建环境缺失，不是项目断言失败。获得本次构建依赖联网授权后，使用临时缓存完成：

```text
env UV_CACHE_DIR=/tmp/skill-engineering-uv-cache uv build \
  --out-dir /tmp/skill-engineering-license-build

Successfully built skill_engineering-1.0.0.tar.gz
Successfully built skill_engineering-1.0.0-py3-none-any.whl
```

wheel `METADATA` 关键字段：

```text
Name: skill-engineering
Version: 1.0.0
Author: 艾笑
License: Apache-2.0
License-File: LICENSE
License-File: NOTICE
```

根 `LICENSE` 与 2026-07-18 下载的 Apache 官方 `LICENSE-2.0.txt` 字节级比较一致，附录占位符保持原样。实际版权人通过 `NOTICE` 和包作者元数据声明，避免把项目定制内容误称为官方许可证原文。

wheel 实际包含 `skill_engineering-1.0.0.dist-info/licenses/LICENSE` 和 `.../NOTICE`；sdist 包含根 `LICENSE`、`NOTICE`、`CITATION.cff`、`TRADEMARKS.md` 以及本轮 Spec/Plan/ADR/测试。
