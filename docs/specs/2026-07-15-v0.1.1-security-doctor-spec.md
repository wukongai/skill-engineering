# Skill Engineering v0.1.1 Security Doctor Spec

## 背景

现有 Doctor 已能检查提示注入、凭证访问、网络传输、真实 secret 和 credentialed provider contract，但脚本安全仍以正则和信号共现为主。与 NVIDIA SkillSpector 的安全分析能力对比后，当前最明确的缺口是 Python 语法级行为识别和标准化 CI 报告。

详细对比见 `docs/research/2026-07-15-nvidia-skillspector-comparison.md`。

## 失败模式

1. import alias、嵌套调用或多行调用让文本正则漏掉危险执行。
2. `input()`、环境变量、网络响应或文件内容流入 `exec`/`eval`/shell sink 时，Doctor 只看到分散信号，不能报告数据流关系。
3. CI 和 IDE 只能消费自定义 JSON，不能使用 SARIF 的规则、级别和位置模型。

## 根因层级

- AST 与 source-to-sink：`script`。
- SARIF 输出：`interface`。
- 规则说明与回归夹具：`test` / `style`，不得把细节堆入根 `SKILL.md`。

## 范围

### Python AST 安全规则

Doctor 扫描 `scripts/**/*.py`，新增：

- `SEC108`：`exec`、`eval` 等动态代码执行。
- `SEC109`：非静态 `compile`、`__import__` 或 `importlib.import_module` 动态加载。
- `SEC110`：`os.system`、`os.popen`，或 `subprocess` 配合 `shell=True` 的不安全 shell 执行。
- `SEC111`：用户输入、环境变量、网络输入或文件读取结果直接或经局部变量流入代码/命令执行 sink。

每个 finding 必须包含 rule id、profile 级别、security layer、相对或可追踪路径、行号、说明和修复建议。语法错误文件由现有 lint/执行验证负责，不因 AST 解析失败制造新的安全结论。

### SARIF 2.1.0

- `doctor` 与 `audit` 增加 `--format text|json|sarif`。
- 现有 `--json` 保持兼容，等价于 `--format json`。
- SARIF `runs[0].tool.driver` 标识 Skill Engineering 和版本。
- 每个 Doctor issue 映射为一个 result；`FAIL -> error`，`WARN -> warning`。
- result 包含 rule id、message、artifact URI、可用时的起始行，以及 layer/profile properties。
- 没有 issue 时仍输出合法的空 results SARIF。

## 安全与误报边界

- 不执行被扫描脚本。
- 不读取 `scripts/` 之外的任意 Python 代码作为行为结论。
- 不引入网络请求、YARA、LLM provider 或新运行时依赖。
- 不用 AST finding 宣称实际恶意意图；它只表示需要修复或人工复核的行为风险。
- 不新增无理由 suppression；waiver/baseline 另行设计审计和到期契约。

## 非目标

- 完整跨函数、跨文件或运行时污点分析。
- JavaScript/TypeScript AST。
- OSV、YARA、远程 Git/zip 下载、MCP tool poisoning。
- SkillSpector 的风险分公式或 LLM 语义复核。

## 兼容性

- `doctor_skill()` 和 `DoctorResult` 现有 Python API 保持兼容。
- 默认文本输出、退出码与 `--json` 保持兼容。
- 新规则只作用于 `scripts/**/*.py`。
- 包的最低 Python 版本继续为 3.10。

## 验收标准

1. import alias 下的危险执行可被识别。
2. 安全的 `subprocess.run(["tool", "--help"], check=True)` 不被 `SEC110` 误报。
3. `shell=True`、`os.system`、动态 `exec/eval` 被定位到正确行。
4. `input()`/环境变量/文件或网络来源经局部变量进入执行 sink 时触发 `SEC111`。
5. 普通业务脚本和合法 provider fixture 不新增误报。
6. SARIF 有效映射规则、级别、位置和 properties；空报告结构有效。
7. CLI 默认文本、`--json` 和 `--format json` 结果兼容。
8. README、Doctor 标准、Changelog、Task、Sprint 和 Backlog 与实现一致。
9. pytest、Ruff、Skill Engineering production Doctor、官方 Skill validation、凭证 lint 和 `git diff --check` 全部通过。
