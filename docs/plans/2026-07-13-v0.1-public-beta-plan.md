# Skill Engineering v0.1 Public Beta 实施计划

状态：隔离候选已完成并通过本地验证，等待应用真实仓库。

对应 Spec：`docs/specs/2026-07-13-v0.1-public-beta-spec.md`

## 0. 收紧能力探索和产物判断

- 新需求先自查已有 Skill、Script、Plugin/runtime、项目规则和相似实现。
- 关键答案缺失时返回 `needs_discovery`,不默认创建 Skill。
- 默认反馈一次只显示一个关键问题。
- 信息足够后比较 2-3 个相关产物方向,再进入架构和治理。

验证：decision tests 覆盖模糊需求、一次性任务、runtime 直达和完整答案创建 Skill。

## 1. 建立仓库自举治理

- 新增产品定位、工程宪章和文档索引。
- 新增 Roadmap、Backlog、当前 Task、版本规则和 Changelog。
- 新增 ADR、Sprint 和 Daily Log 目录及首份记录。
- 更新 AGENTS，使功能开发必须对齐上述事实源。

验证：文档链接存在，当前版本、后续 Backlog 和非目标没有冲突。

## 2. 强化 Skill Guide 的复杂/商业 Skill 引导

- 新增开发治理 reference 和 stage。
- 根 SKILL.md 只增加路由，不塞入详细项目管理流程。
- contract 增加商业治理输出、delegate 和禁止行为。
- 明确简单 Skill 不需要完整项目文档，按治理等级渐进披露。

验证：根入口保持轻量；Doctor 无新增 FAIL/WARN；触发描述覆盖创建、修改、重构、防腐化和版本演进。

## 3. 收紧创建 Apply 契约

- `create` 增加 `--plan`。
- Preview 和 Apply 分离；直接 `--apply` 被拒绝。
- 创建计划增加不可变 plan hash。
- Apply 前验证 plan hash、operation 和目标指纹。
- 写入后运行 lint + team Doctor；失败时删除整个新目标。
- production 额外记录 production Doctor，但允许预期的证据缺口保持 incomplete。
- 修正创建后用户反馈，区分未写入、结构完成和生产证据未完成。

验证：为 preview、same-plan apply、直接 apply 拒绝、plan 漂移、postflight 清理和 production incomplete 增加测试。

## 4. 建立开源 Public Beta 外壳

- 更新 README 的定位、状态、安装、快速体验、能力边界和路线链接。
- 新增 CONTRIBUTING、SECURITY 和 GitHub CI。
- 保持版本 `0.1.0`；只有全部发布门禁通过后才标记发布候选。

验证：CI 命令可在本地等价执行；仓库无凭证和本机私有路径。

## 5. 完整验证

- 定向运行 scaffold、CLI、Skill Guide 测试。
- 运行完整 pytest 和 Ruff。
- 运行 Skill Guide production Doctor。
- 运行凭证 lint 和 `git diff --check`。
- 在 `/private/tmp` 执行真实 preview -> plan apply -> postflight E2E。
- 检查 README、版本、Changelog、Roadmap、Task 和 Sprint 状态一致。

## 6. 发布确认点

- 生成真实仓库 maintenance plan，展示文件增删改、复杂度和门禁结果。
- 用户明确确认后，应用同一份未漂移计划到真实仓库。
- 应用后重复完整验证。
- commit 和 push 分别停在明确确认点；不自动公开发布。
