# Skill Engineering v2.0 Architecture Guardian 实施计划

状态：In Progress，当前 Sprint 先完成 Blueprint/IR 契约

对应 Spec：`docs/specs/2026-07-16-v2.0-architecture-guardian-spec.md`

## Phase 1：Blueprint/IR 契约（当前开发）

- 定义 `blueprint.schema.json` 的 schema version、三轴字段、unknown/legacy 表达和扩展点；
- 增加 Python dataclass/校验器与确定性 JSON 序列化；
- 为 atomic/router/orchestrator/adapter 建立最小 fixture；
- 记录 schema 选择、兼容和迁移原则到 ADR；
- 只读生成 preview，不接入 apply。

验收：schema validation、稳定 fingerprint、未知字段保留、负例和敏感信息 lint 通过。

## Phase 2：Inventory extractor

- 从入口、references、scripts、contract、tests 提取组件和边界；
- 识别 delegate、状态、外部副作用、依赖和治理证据；
- 对无法确定的事实输出 `unknown` 与证据路径，不让模型猜测。

## Phase 3：Guardian checks

- 依赖图、重复职责、route collision；
- context budget、入口厚度、状态/副作用/治理适配度；
- baseline/candidate semantic diff；
- findings、severity、remediation 和 regression contract。

## Phase 4：Plan integration

- guardian finding 转换为 create/improve preview plan；
- 复杂度 delta、删除/迁移清单、apply 前 fingerprint；
- 与现有 maintenance history、verify、undo 对齐。

## Phase 5：Migration and release

- 自动生成拆分、迁移、压缩和废弃候选；
- 真实 baseline/holdout/high-risk/negative-transfer 验证；
- 2.0 RC、兼容迁移和发布记录。

## 恢复边界

Phase 1 失败时只撤销 Blueprint/IR 新增文件，不影响 0.1.x Doctor、evaluate、improve 和 release；后续阶段未通过证据门禁时留在 preview，不升级为 active。
