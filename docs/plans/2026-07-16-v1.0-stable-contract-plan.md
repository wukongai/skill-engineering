## 阶段

1. **Contract inventory**：盘点所有 CLI、JSON、contract、plan、record 和 state schema，标注 stable/preview/internal。
2. **Compatibility**：增加 schema version、旧格式读取、迁移错误和 deprecation policy。
3. **Packaging**：建立 wheel、干净环境安装、Python 矩阵和 smoke 测试。
4. **Evidence**：固定 release evidence 模板、独立评审和回滚检查。
5. **Documentation**：对齐 README、版本源、Changelog、发布日志、Task、Sprint 和支持范围。
6. **RC**：运行完整门禁，生成 RC 记录；用户确认后才允许 tag/publish。

## 风险与恢复

- 旧 JSON 字段语义漂移：先增加兼容读取和 fixture，再改变输出。
- 包版本与文档不一致：发布前使用单一版本检查脚本。
- 证据不足：只能标记 preview/incomplete，不得宣称 stable。
- 发布失败：保留上一稳定版本和可撤销 release record。
