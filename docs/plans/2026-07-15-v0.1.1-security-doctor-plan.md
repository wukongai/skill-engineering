## 2. 增加独立 AST 安全分析模块

- 使用 Python 标准库 `ast`，建立 import alias 和调用名解析。
- 在单个函数/模块内跟踪外部输入到执行 sink 的直接和局部变量传播。
- 返回独立、可测试的 finding，不直接耦合 CLI 格式。
- Doctor 把 finding 映射为 `SEC108-SEC111` issue，并按 profile 决定 WARN/FAIL。

验证：新增专用单元测试覆盖 safe、alias、dynamic execution、unsafe shell、direct taint、variable taint 和 syntax-error fallback。

## 3. 增加 SARIF 输出

- 从 `DoctorResult` 构建最小合法 SARIF 2.1.0 dict。
- 增加 `format_sarif()` 和统一格式选择。
- CLI 增加 `--format`，保留 `--json` 兼容别名。

验证：单元测试断言 schema/version、driver、rules、result level、位置、properties 和空 results；CLI 测试覆盖三种格式。
