# Sprint：v0.1.1 Security Doctor

状态：Completed，Unreleased

日期：2026-07-15

## 目标

吸收 NVIDIA SkillSpector 中与现有产品边界最契合的安全工程优点，为 Doctor 增加 Python AST 行为风险识别和 SARIF 2.1.0 输出，同时保持本地、确定性、provider-neutral 和低依赖。

## 进入范围

- SkillSpector 对比研究与采用矩阵；
- `SEC108-SEC111` Python AST/data-flow 规则；
- `doctor/audit --format sarif`；
- CLI 兼容、文档、回归测试和发布证据；
- Skill Guide Doctor reference 的候选/计划/Apply 闭环。

## 不进入范围

- LLM 语义复核、YARA、OSV、远程下载；
- 完整跨函数/跨文件污点分析；
- suppression/waiver；
- MCP 专项扫描；
- commit、push、tag 或公开发布。

## 风险

- 把所有 subprocess 使用误判为恶意；通过只拦截动态执行、`shell=True` 和明确 source-to-sink 降低误报。
- 照搬完整扫描器导致依赖和边界膨胀；本轮只使用 Python 标准库。
- SARIF 与 JSON 产生不同结论；两者必须由同一个 `DoctorResult` 生成。
